# PRD Wizard Lite 🧙‍♂️

이 스킬은 외부 도구나 라이브러리 설치 없이, 순수하게 대화와 문서 작성을 통해 제품 기획(PRD)을 돕습니다.

## 핵심 가치
- **대화형 요구사항 도출**: 사용자가 미처 생각하지 못한 부분까지 질문으로 이끌어냅니다.
- **표준화된 문서**: 엔지니어가 바로 이해할 수 있는 수준의 상세 PRD를 작성합니다.
- **유연성**: 특정 도구(TaskMaster 등)에 종속되지 않고 프로젝트 상황에 맞춰 문서를 구성합니다.

## 위자드 프로세스 (Phase 1~5)

### Phase 1: 핵심 컨셉 발견 (Discovery)
아래의 5가지 핵심 질문을 통해 기능의 본질을 정의합니다:
1. **문제 정의**: 이 기능이 해결하려는 사용자의 고통은 무엇인가요?
2. **타겟 사용자**: 이 기능을 주로 사용할 사람은 누구인가요?
3. **핵심 가치**: 이 기능이 제공하는 가장 큰 혜택은 무엇인가요?
4. **성공 지표**: 기능이 성공적이라는 것을 어떻게 측정할까요?
5. **제약 사항**: 기술적, 시간적, 정책적 제약이 있나요?

### Phase 2: 상세 요구사항 도출 (Deep Dive)
기본 컨셉이 잡히면 기술적인 부분을 파고듭니다:
- **사용자 시나리오**: 사용자가 어떤 흐름으로 기능을 사용하게 되나요?
- **데이터 모델**: 어떤 정보가 저장되고 관리되어야 하나요?
- **UI/UX 고려사항**: 화면 구성이나 사용자 경험에서 중요한 포인트는?
- **에지 케이스**: 예외적인 상황(네트워크 오류, 잘못된 입력 등)에서는 어떻게 동작해야 하나요?

### Phase 3: PRD 초안 작성 (Drafting)
수집된 정보를 바탕으로 `PRD.md` 문서를 작성합니다. 포함 섹션:
- Executive Summary
- Goals & Success Metrics
- User Stories & Acceptance Criteria
- Functional Requirements (REQ-001...)
- Technical Considerations & Arch
- Out of Scope

### Phase 4: 품질 검증 (Quality Check)
작성된 PRD를 스스로 리뷰합니다:
- 모든 요구사항이 테스트 가능한가?
- 비즈니스 로직에 모순은 없는가?
- 누락된 에러 케이스는 없는가?

### Phase 5: 최종 확정 및 테스크 도출
사용자의 피드백을 반영하여 PRD를 확정하고, 구현을 위한 개발 태스크 목록을 생성합니다.

## 실행 가이드
- 사용자가 "기획 도와줘", "PRD 작성해줘", "기능 추가할 건데 고민해줘"라고 하면 이 스킬을 활성화합니다.
- 질문은 한 번에 너무 많이 던지지 않고, 맥락에 따라 2~3개씩 나누어 대화의 호흡을 유지합니다.
